### 2. 策略提示 - 全面重构为智能分级系统

#### 必显示提示（4条）

**1. 获客效率分级** 🚀✅⚠️🔴🚨
```
LTV/CAC > 3    → 🚀 优秀！建议大胆放量30-50%
LTV/CAC 2-3    → ✅ 健康，可稳步放量
LTV/CAC 1.5-2  → ⚠️ 偏紧，小步放量+优化
LTV/CAC 1-1.5  → 🔴 盈亏平衡，暂停放量
LTV/CAC < 1    → 🚨 亏损，立即停止投放
```

**2. 现金流风险等级** 🚨⚠️✅
```
净现金流 < 0         → 🚨 提现超过充值
覆盖月数 < 1个月     → 🚨 高风险
覆盖月数 1-2个月     → ⚠️ 偏紧
覆盖月数 > 2个月     → ✅ 健康
```

**3. 成本优化优先级** 🚨⚠️✅
```
总成本率 > 85%  → 🚨 成本过高（列出Top2成本项）
总成本率 70-85% → ⚠️ 需优化（列出Top2成本项）
总成本率 < 70%  → ✅ 结构健康
```

**4. 12个月累计**
- 亏损 → 建议小规模验证
- 盈利 → 可用于融资基准

#### 条件触发提示（最多3条）

**5. 提现率异常**
- \> 80% → 玩家赢太多，检查RTP
- < 50% → 门槛太高，影响口碑

**6. 支付成功率过低**
- < 75% → 每月损失XX，建议多网关

**7. 用户质量问题**
- 首充转化 < 15% → 严重偏低
- 付费率 < 2% → 免费用户拉高成本

**8. 收入结构风险**

## 🎯 新增的核心能力

### 1. 智能分级系统
不再是简单的"好/坏"二分法，而是5级细分：
- 🚀 优秀（>3x）
- ✅ 健康（2-3x）
- ⚠️ 偏紧（1.5-2x）
- 🔴 临界（1-1.5x）
- 🚨 危险（<1x）

### 2. 优先级排序
自动识别最重要的成本项：
```javascript
const costItems = [促销, 获客, 固定, 供应商];
排序后取Top2，给出具体优化建议
```

### 3. 条件触发逻辑
只有真正有问题的指标才触发预警，避免信息过载

### 4. 可执行的建议
不是泛泛而谈，而是具体的数字和步骤：
- "降低首充门槛至₹100"（不是"优化首充"）
- "提高营销预算30-50%"（不是"可以放量"）
- "接入3-5家支付网关"（不是"优化支付"）
